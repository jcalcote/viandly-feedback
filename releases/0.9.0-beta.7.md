# Viandly 0.9.0-beta.7

Released: July 23, 2026

This closed-beta update makes scanner-delivered recipe review more reliable, corrects AI image
capability detection, and polishes several frequently used interface details.

## Highlights

- Corrected AI image-capability testing so valid vision models are no longer reported as text-only
  because of an unrelated structured-response error.
- Added a clear path for verifying image support when an existing AI connection has not yet been
  tested for vision.
- Preserved local OCR as an intentional scanner-processing choice even when AI vision is available.
- Centered scanner-delivery status labels and aligned scanner review actions.
- Changed the signed-in home Inbox summary to show **All caught up** when no work is waiting.
- Realigned recipe image-upload controls and moved file-format guidance beneath the complete control
  row.
- Expanded scanner setup Help with a tested appliance workflow, including scanner administration,
  Gmail SMTP with an app password, address-book setup, and practical scan settings.

## Tester Guidance

AI provider connections configured before scanner vision detection was introduced need one
capability check:

1. Open **Account Settings > AI Provider**.
2. Select **Edit**.
3. Select **Test selected model**.

New AI connections and later model changes perform capability testing during their normal setup.
Local OCR remains available for clean scans and for models that do not accept images.
