# Viandly 0.9.0-beta.4

Released: July 21, 2026

This closed-beta update repairs AI provider setup so model discovery and account validation reflect
the capabilities Viandly actually needs before users begin an AI-assisted workflow.

## Highlights

- Fixed model discovery so users can load provider models before selecting or manually entering a
  model.
- Preserved an unsaved API key while moving through model discovery and selected-model testing.
- Added a dedicated selected-model test that sends a small structured text request through the same
  provider API family used by Viandly features.
- Added validation before saving new connections, provider switches, credential rotations, and
  model changes so a failed replacement cannot overwrite a working connection.
- Added clearer distinctions between rejected credentials, denied model access, incompatible
  models, temporary rate limits, exhausted quota or spending limits, and provider outages.
- Applied the shared provider-error handling to AI import, import repair, recipe development, and
  cookbook design.

## Compatibility

This release does not change the relational database schema or REST API version. Existing AI
provider connections remain valid and are retested only when a user changes the saved model or
credential.
