# Viandly 0.9.0-beta.5

Released: July 22, 2026

This closed-beta update adds AI-enhanced visual recipe extraction for scanner-delivered images and
image-only PDFs, while preserving searchable PDF and local OCR alternatives.

## Highlights

- Added **Analyze with AI** for entitled scanner deliveries when the configured provider model has
  verified image-input support.
- Added visual recipe comprehension for full pages, multi-column layouts, recipes continued across
  pages, surrounding editorial material, missing section headings, and common recognition defects.
- Added OpenAI-compatible, Anthropic, and Google visual-extraction support using each user's own AI
  account and credential.
- Added bounded background page preparation, PDF rasterization, retries, per-account admission,
  idempotent draft creation, and cleanup of temporary scanner objects.
- Preserved searchable PDF embedded-text extraction as the preferred fast path when useful text is
  already present.
- Preserved browser-local OCR as a privacy and recovery option for manual region selection or when
  users do not want to send page images to an AI provider.
- Added separate image-input capability testing in AI Provider settings so text-only models remain
  usable without being recommended for scanner vision.
- Added provider disclosure, background progress, and scanner guidance covering resolution, color,
  searchable PDFs, OCR text layers, file formats, duplex scanning, cropping, privacy, and cost.
- Documented the current one-delivery, one-draft boundary: an AI-analyzed delivery should contain
  one independent recipe.

## Compatibility

This release adds image-capability metadata to existing AI provider connections and new scanner
extraction and background-job types. Existing connections remain valid and begin with unknown image
capability until their selected model is tested.
